## 🎯 Flagship Projects

### 🧾 AI-Powered Resume Builder — GenAI Application
> An agentic resume-tailoring tool: parses a resume, compares it against a job description, and rewrites content for fit — not a wrapper, a full pipeline.

| | |
|---|---|
| **Problem** | Manually tailoring resumes per job application is slow and inconsistent |
| **Approach** | LangGraph multi-step agent + Groq (`llama-3.1-8b-instant`) + Streamlit UI, built across 9 development phases |
| **Engineering depth** | Fixed JSON parsing failures, LLM token rate limits, DOCX table-reading bugs, and session-state persistence — the problems that surface once you move past a toy demo |
| **Stack** | `LangGraph` `Groq API` `Streamlit` `Python` |

<br>

### ⚙️ Predictive Maintenance — Vibration Anomaly Detection
> Reproduced a published research pipeline, then beat it with a deep learning model.

| | |
|---|---|
| **Problem** | Detect early-stage mechanical failure from exhaust-fan vibration sensor data |
| **Approach** | Reproduced Amruthnath & Gupta (2018)'s unsupervised pipeline — PCA T²/SPE, K-Means, Hierarchical Clustering, Fuzzy C-Means, GMM — then extended it with Isolation Forest, One-Class SVM, MLP Autoencoder, and an **LSTM Autoencoder** |
| **Result** | LSTM Autoencoder identified as the strongest-performing model |
| **Deliverables** | Full notebooks, 19-slide technical deck, 10-paper literature review with gap analysis |
| **Stack** | `TensorFlow` `Keras` `Scikit-learn` |

<br>

### 📡 Real-Time Predictive Maintenance Framework
> Moved the above problem from batch to streaming.

| | |
|---|---|
| **Problem** | Batch-only pipelines can't catch equipment failure as it happens |
| **Approach** | Kafka + Spark Structured Streaming feeding an LSTM model, applied to NASA's CMAPSS turbofan degradation dataset |
| **Stack** | `Kafka` `PySpark` `LSTM` |
